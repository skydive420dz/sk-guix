#+title: sk-guix

Personal Guix channel for packages that intentionally move faster than Guix
main on GuixPC.

Initial policy:

- build small, realistic packages from source;
- use binary packaging only when source builds are impractical;
- keep recipes narrow and auditable.

Current decisions:

- Keep Fish on mainstream Guix for now.
- Evaluate Kitty latest in this channel, but do not wire it into
  =guix-dotfiles= until explicitly chosen.

Current WIP packages:

- =fish-latest=
- =kitty-latest=

Status:

* Fish

- Fish 4.8.0 is not a clean version bump over Guix main's Fish package.
- It adds fixed Git dependencies for Fluent crates and additional Rust crates
  beyond the inherited cargo input set.
- The recipe currently evaluates but does not build. The first unresolved crate
  after vendoring the Git dependencies is =unic-langid=.
- Do not wire this into =guix-dotfiles= until =guix build fish-latest= passes.

Decision: keep using Guix main's Fish package.

* Kitty

- Guix main: =kitty 0.46.2=.
- Evaluated upstream: =kitty 0.47.4=.
- Source commit: =a685af8c351bc4521be17e939eb619b76dbcd646=.
- Source hash: =1m8sn8hs63qw8n3hvn07pqmnd4grqfr59pxgwa4jq7ivd1nrcfsh=.
- New overlay Go dependencies:
  - =go-github-com-emmansun-base64 0.9.0=
  - =go-github-com-sgtdi-fswatcher 1.3.0=

The recipe inherits Guix's Kitty package and applies narrow test skips for
build-container assumptions:

- =kitty_tests/dnd_kitten.py=: Guix already disables the graphics test module
  because it needs display-server access; Kitty 0.47.4's DnD kitten test imports
  that disabled module.
- =tools/utils/machine_id/api_test.go=: assumes =/etc/machine-id= exists, which
  is false inside the Guix build container.

Acceptance:

- =guix build -L ~/Projects/sk-guix kitty-latest --no-grafts= passes.
- The built binary reports =kitty 0.47.4 created by Kovid Goyal=.

Decision: Kitty latest is viable for evaluation, but not wired into GuixPC yet.
